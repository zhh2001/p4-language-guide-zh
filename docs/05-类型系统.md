# 05 · 类型系统

> 本章目标：把 P4_16 的所有数据类型一次性讲透——从最底层的位串，到最关键的 `header`、`struct`、`header_union`、报头栈。这是后面所有章节的基石。

P4 的类型系统相对 C 做了重大简化：没有指针、没有联合体（C 意义上的 union，P4 有的是 `header_union`）、没有浮点、没有运行时可变长度数组。它增加的新类型是**为了精确描述网络报文的布局**。

## 5.1 基本类型一览

| 类型 | 说明 |
| ---- | ---- |
| `void` | 空类型，与 C 同 |
| `bool` | 布尔，`true` / `false` |
| `error` | 错误类型（见 5.3） |
| `string` | 字符串（仅字面量） |
| `match_kind` | 表匹配类型（见 08 章） |
| `int` | 任意精度整数（仅字面量 + 常量表达式） |
| `bit<W>` | 无符号位串，宽度 W |
| `int<W>` | 有符号整数，宽度 W，补码表示 |
| `varbit<W>` | 动态宽度位串，最大宽度 W |

## 5.2 `void`

和 C 的 `void` 基本一致——表示"没有返回值"或"没有数据"。

## 5.3 `error`

所有 `error` 常量不管在哪里定义，都会被收编进同一个全局 `error` 命名空间：

```p4
error { PacketTooShort, IPv4BadVersion }
```

看起来像 enum，但：

- 你**只能**添加 error 成员，不能给具体值
- 编译器会自动聚合 —— 多个 `error { ... }` 声明最终合并成一个

典型用法：在 parser 里调用 `verify(cond, error.IPv4BadVersion)`（见第 06 章）。

## 5.4 `match_kind`

用于声明"表的匹配方式"名字。规范的 `core.p4` 里已经定义了：

```p4
match_kind { exact, ternary, lpm }
```

**只能在架构描述文件里声明新的 `match_kind`。** 普通程序员不用管。

## 5.5 `bool` 与 `string`

- `bool`：`true` / `false`。**不能与整数互相赋值**（需要显式转换）。
- `string`：仅字面量；不能声明变量。常见于注解。

## 5.6 整数类型

### 5.6.1 无符号 `bit<W>`

P4 的核心数据类型。宽度可以是常量表达式：

```p4
const bit<32> X   = 10;
const bit<(X+2)> Y = 15;  // 12 位，宽度是表达式必须加括号
```

> [!TIP]
> `bit` 等价于 `bit<1>`。

位编号约定：`bit<128>` 的第 0 位是**最低有效位**，第 127 位是**最高有效位**。

### 5.6.2 有符号 `int<W>`

补码表示，最高位是符号位。P4 禁止大量"模糊"的运算：不能把 `bit` 和 `int` 直接相加、不能用有符号数做位移量、比较时也不会自动提升。详见 [04.15 类型转换](./04-语法基础.md#415-显式与隐式类型转换)。

### 5.6.3 动态宽度 `varbit<W>`

`varbit<W>` 表示"最多 W 位"的位串。典型用途：IPv4 options、TCP options。

```p4
header IPv4_options_h {
    varbit<320> options;   // 最多 320 位
}
```

对 `varbit` 能做的操作非常有限——只能参与 `extract` 和 `emit`，**不能赋值、不能做位运算**。

### 5.6.4 `int`（无限精度）

仅用于**字面量**和**纯字面量组成的常量表达式**：

```p4
const int A = 5;
const int B = 2 * A;
const int C = B - A + 3;
```

如果拿 `int` 当函数参数，必须是无方向的。

## 5.7 `enum` 枚举

```p4
// 无底层类型：不能序列化，只能比较相等
enum Suits { Clubs, Diamonds, Hearts, Spades }

// 带底层类型（"可序列化"枚举）：可以出现在报头里
enum bit<16> EtherType {
    VLAN = 0x8100,
    QINQ = 0x9100,
    MPLS = 0x8847,
    IPV4 = 0x0800,
    IPV6 = 0x86DD
}
```

带底层类型的 enum 和底层类型之间 **可以隐式转换**（单向：enum → 底层类型）。

## 5.8 `header` —— 最重要的类型

`header` 是 P4 对 **网络报头** 的一等抽象。它像 struct，但多了一个隐藏的**有效位（validity bit）**：

- `isValid()` 返回当前是否有效
- `setValid()` / `setInvalid()` 改变有效位

### 5.8.1 声明

```p4
header Ethernet_h {
    bit<48> dstAddr;
    bit<48> srcAddr;
    bit<16> etherType;
}
```

字段类型只能是：`bit<W>`、`int<W>`、带底层类型的 enum、`bool`。**不能嵌套 header**。结构体可以作为"命名字段组"内嵌：

```p4
struct ipv6_addr {
    bit<32> a0; bit<32> a1; bit<32> a2; bit<32> a3;
}
header IPv6_h {
    bit<4>  version;
    bit<8>  trafficClass;
    bit<20> flowLabel;
    bit<16> payloadLen;
    bit<8>  nextHdr;
    bit<8>  hopLimit;
    ipv6_addr src;
    ipv6_addr dst;
}
```

### 5.8.2 空报头

```p4
header Empty_h { }   // 合法，但仍有有效位
```

### 5.8.3 有效位的重要性

刚声明一个 header 变量时，有效位自动是 `false`：

```p4
Ethernet_h eth;       // eth.isValid() == false
eth.dstAddr = 0;      // 仍然 invalid，但可以赋值
eth.setValid();       // 现在 valid 了
```

> [!WARNING]
> 只有 valid 的 header 才会被 `emit` 输出。写了但忘记 `setValid()`——是 P4 最常见的 bug 之一。

### 5.8.4 字面量初始化

```p4
header H { bit<32> x; bit<32> y; }
H h;
h = { 10, 12 };          // 按字段顺序
h = { y = 12, x = 10 };  // 按名称
h = {#};                 // 无效报头
```

### 5.8.5 比较

```p4
h1 == h2     // 仅当类型相同且有效位相同且所有字段相等
h == {#}     // 等价于 !h.isValid()
```

### 5.8.6 方法

| 方法 | 作用 |
| ---- | ---- |
| `hdr.isValid()` | 返回有效位 |
| `hdr.setValid()` | 有效位置 `true` |
| `hdr.setInvalid()` | 有效位置 `false` |

## 5.9 报头栈 `header stack`

表示一个 **相同类型 header 的定长数组**，常用于 MPLS、VLAN、IPv6 扩展头等。

```p4
header Mpls_h {
    bit<20> label;
    bit<3>  tc;
    bit<1>  bos;
    bit<8>  ttl;
}

Mpls_h[10] mpls;   // 最多 10 层 MPLS
```

### 5.9.1 栈内隐藏字段

概念上报头栈就是 `data[N]` + 一个 `nextIndex` 计数器 + `size`。初始 `nextIndex = 0`。

### 5.9.2 合法操作

| 表达式 | 语义 |
| ------ | ---- |
| `hs[i]` | 下标访问（可读可写） |
| `hs.size` | 栈大小（常量） |
| `hs.next` | `hs[nextIndex]`，**仅在 parser 中**；越界 reject |
| `hs.last` | `hs[nextIndex-1]`，**仅在 parser 中**；越界 reject |
| `hs.lastIndex` | `nextIndex - 1` |
| `hs.push_front(n)` | 整体后移 n 位，前 n 个置 invalid |
| `hs.pop_front(n)`  | 整体前移 n 位，后 n 个置 invalid |
| `hs == hs2` | 比较所有元素，`nextIndex` 不参与 |

### 5.9.3 初始化

```p4
H<bit<32>>[3] s = { {0, 1}, {2, 3}, (H<bit<32>>){#} };
H<bit<32>>[3] s2 = { {0, 1}, {2, 3}, ... };   // 剩余用默认值
```

## 5.10 报头联合体 `header_union`

同一段内存可能是几种不同报头中的任意一种。典型用法：同一个字段既可能是 IPv4 也可能是 IPv6。

```p4
header_union IP_h {
    IPv4_h v4;
    IPv6_h v6;
}
```

### 5.10.1 语义要点

- `u` 未初始化时，所有成员有效位都是 `false`
- 给某个成员赋值会 **自动使其他成员 invalid**
- `u.hi.setValid()` 会把 **其他成员全部** invalid
- `u.isValid()` 只有**某一个** 成员 valid 时返回 true

### 5.10.2 在 parser 里的巧用

```p4
parser P(packet_in b, out Parsed_packet p) {
    state start {
        b.extract(p.ethernet);
        transition select(p.ethernet.etherType) {
            0x0800: parse_ipv4;
            0x86DD: parse_ipv6;
        }
    }
    state parse_ipv4 { b.extract(p.ip.v4); transition accept; }
    state parse_ipv6 { b.extract(p.ip.v6); transition accept; }
}
```

## 5.11 `struct`

和 C 的 struct 语义一致——字段不限定类型，没有有效位。

```p4
struct metadata_t {
    bit<9>  orig_ingress_port;
    bit<16> l4_src_port;
    bool    is_multicast;
}
```

用途：

- 在块之间传递**解析出来的所有报头**（`headers` 结构）
- 在块之间传递**流水线元数据**（`metadata`）

初始化语法同 header，但没有 `{#}`。

## 5.12 `tuple`

**匿名**的、字段**无名字**的多元组：

```p4
tuple<bit<32>, bool, bit<16>> t;
```

常见场景：`hash(output, algo, base, { f1, f2, ... }, max)` 的输入。

## 5.13 `list`

`list<T>` 表示元素类型全相同的列表。主要用于字面量上下文（如 `select` 的标签集合）：

```p4
list<bit<8>> = { 1, 2, 3, 4 }
```

## 5.14 `enum bit<W>`（可序列化枚举）回顾

再次强调——放进 header 里的字段类型必须是 **有底层类型的 enum**。例如：

```p4
enum bit<16> EtherType {
    IPV4 = 0x0800,
    ARP  = 0x0806,
}

header Ethernet_h {
    bit<48>   dst;
    bit<48>   src;
    EtherType etype;   // OK
}
```

普通 `enum`（无底层类型）只能用在变量里，不能出现在 header 里。

## 5.15 `extern`

外部对象——由目标（BMv2、Tofino）提供的资源：计数器、寄存器、哈希、校验和等。详见 [12 · 外部对象](./12-外部对象Extern.md)。

## 5.16 `parser` / `control` / `package` 类型

这三个是 P4 独有的 **可调用对象类型**。声明的是"接口"，之后用 `{ ... }` 给出实现。详见 [06 · Parser 解析器](./06-Parser解析器.md)、[07 · 控制块与动作](./07-控制块与动作.md)、[10 · 架构与包](./10-架构与包.md)。

## 5.17 默认值

```p4
bit<W>      → 0
int<W>      → 0
int         → 0
bool        → false
error       → error.NoError
string      → ""
varbit<W>   → 零长位串
header      → invalid
header_union → 所有成员 invalid
header stack → 所有元素 invalid，nextIndex = 0
```

**没有** 默认值的类型：`match_kind`、函数类型、extern、parser、control、package、set。

## 5.18 用 `...` 做默认初始化

```p4
struct S { bit<32> b32; bool b; }
enum int<8> N0 { one = 1, zero = 0, two = 2 }
enum N1 { A, B, C }

struct T { S s; N0 n0; N1 n1; }

N0 n0 = ...;                // 0
N1 n1 = ...;                // N1.A
S  s1 = { 1, ... };         // { 1, false }
S  s2 = { b = true, ... };  // { 0, true }
T  t0 = ...;                // { {0,false}, 0, N1.A }
T  t3 = { ..., n0 = 2 };    // ❌ ... 必须在最后
```

对 header：

```p4
header H { bit<16> f1; bit<8> f2; }

H h1 = ...;                 // invalid header
H h2 = { f2 = 5, ... };     // valid，{ 0, 5 }
H h3 = { ... };             // valid，{ 0, 0 }
```

## 5.19 `typedef` vs `type`

两个类型别名机制，**有本质区别**：

```p4
typedef bit<32> IPv4Addr;   // 透明别名：可与 bit<32> 互换
type    bit<32> NodeId;     // 强类型：需要显式转换
```

使用示例：

```p4
IPv4Addr ip = 0;
bit<32>  x  = ip;        // OK

NodeId   id = (NodeId)32w0;  // 必须显式转换
bit<32>  y  = (bit<32>)id;   // 必须显式转换
```

何时用 `type`？当你想让编译器帮你捕捉"把 `NodeId` 当 `PortId` 用"这种 bug。

## 5.20 本章小结

- P4 类型系统围绕 **描述网络报文** 设计，核心新类型是 `header` / `header_union` / `header stack`
- `header` 自带 **有效位**，忘记 `setValid()` 是高频 bug
- `varbit` 仅用于可变长度 options，能做的操作极少
- 枚举必须有底层类型才能放进 header
- 隐式转换极少；推荐用 `type` 做强类型约束，提前捕获 bug
