# 01 · 环境搭建

> 本章目标：在你自己的机器上准备好 **P4 编译器 + BMv2 软件目标 + Mininet**，并通过自检脚本确认一切就绪。

P4 的学习没有"一键安装"那么简单，因为工具链包含编译器（`p4c`）、软件交换机（BMv2）、拓扑仿真（Mininet）和控制平面（P4Runtime）等多个组件。本章给出三种主流方案，按「推荐程度 × 难度」排序：

| 方案 | 优点 | 缺点 | 适合 |
| ---- | ---- | ---- | ---- |
| Docker 镜像 | 最快，环境隔离 | 体积大（~3 GB），性能略损 | 所有人 |
| apt 包管理器 | 原生速度，系统集成 | 仅 Ubuntu 20.04/22.04 | Ubuntu 用户 |
| 源码编译 | 最新特性 | 耗时、易踩坑 | 开发者、研究者 |

## 方式一：Docker（推荐）

官方维护的镜像包含了 `p4c`、`bmv2`、`p4runtime`、Python 客户端等全部组件。

```bash
# 拉取最新镜像（约 3 GB）
docker pull p4lang/p4app:latest

# 起一个容器，把当前目录挂进去
docker run --rm -it \
    --name p4-dev \
    --privileged \
    -v "$PWD":/workspace \
    -w /workspace \
    p4lang/p4app:latest bash
```

容器内验证：

```bash
p4c --version                  # 应输出 p4c 版本
simple_switch --version        # 应输出 BMv2 版本
which simple_switch_grpc       # /usr/local/bin/simple_switch_grpc
```

> [!TIP]
> **镜像选择**
> - `p4lang/p4app` 偏重运行环境与示例
> - `p4lang/p4c` 仅含编译器，体积较小
> - `opennetworking/p4mn` 集成了 Mininet 与 P4Runtime Shell，适合做控制平面实验

## 方式二：apt 包管理器（Ubuntu 20.04 / 22.04）

P4.org 维护了官方 PPA：

```bash
# 添加 GPG key 和源
. /etc/os-release
echo "deb http://download.opensuse.org/repositories/home:/p4lang/xUbuntu_${VERSION_ID}/ /" \
    | sudo tee /etc/apt/sources.list.d/home:p4lang.list
curl -fsSL "https://download.opensuse.org/repositories/home:p4lang/xUbuntu_${VERSION_ID}/Release.key" \
    | gpg --dearmor | sudo tee /etc/apt/trusted.gpg.d/home_p4lang.gpg > /dev/null

sudo apt update

# 安装三件套
sudo apt install -y \
    p4lang-p4c \
    p4lang-bmv2 \
    p4lang-pi

# 安装 Mininet（用于拓扑仿真）
sudo apt install -y mininet
```

验证：

```bash
p4c --version
simple_switch --version
sudo mn --test pingall       # 应能看到 Mininet 默认拓扑正常 ping 通
```

## 方式三：源码编译（开发者路线）

如果你想跟进 P4 最新语言特性、或要为 `p4c` 添加 backend，必须自行编译。社区最成熟的一键脚本来自 Andy Fingerhut：

```bash
sudo apt install -y git
git clone https://github.com/jafingerhut/p4-guide.git
cd p4-guide
# 选择你的系统版本，例如 Ubuntu 22.04
./bin/install-p4dev-v8.sh |& tee install.log
```

脚本会顺序构建：

1. **protobuf** — P4Runtime 依赖
2. **grpc** — P4Runtime 传输层
3. **PI** — P4 基础设施库
4. **behavioral-model** — BMv2 软件目标
5. **p4c** — 编译器
6. **Mininet** + **ptf** — 仿真与测试框架

脚本完成后，按提示执行：

```bash
source ~/p4setup.bash
```

> [!TIP]
> 把 `source ~/p4setup.bash` 追加到 `~/.bashrc`，以后不必每次手动 source。

> [!WARNING]
> **常见踩坑点**
> - 内存不足：gRPC 编译需 ≥ 4 GB 内存，建议预留 8 GB
> - Python 版本：脚本默认使用系统 Python 3，若有 pyenv 可能冲突
> - 如果中途失败，查 `install.log` 定位，通常是某个 `apt install` 被网络打断，重跑一遍即可

## 自检脚本

准备好后，在本仓库根目录运行：

```bash
cat > /tmp/hello.p4 <<'EOF'
#include <core.p4>
#include <v1model.p4>

struct metadata { }
struct headers { }

parser MyParser(packet_in p, out headers h, inout metadata m, inout standard_metadata_t s) {
    state start { transition accept; }
}
control MyVerify(inout headers h, inout metadata m) { apply { } }
control MyIngress(inout headers h, inout metadata m, inout standard_metadata_t s) {
    apply { s.egress_spec = s.ingress_port; }
}
control MyEgress(inout headers h, inout metadata m, inout standard_metadata_t s) { apply { } }
control MyCompute(inout headers h, inout metadata m) { apply { } }
control MyDeparser(packet_out p, in headers h) { apply { } }

V1Switch(MyParser(), MyVerify(), MyIngress(), MyEgress(), MyCompute(), MyDeparser()) main;
EOF

p4c-bm2-ss --target bmv2 --arch v1model \
    -o /tmp/hello.json /tmp/hello.p4
```

如果最后一步没有报错、且生成了 `/tmp/hello.json`，恭喜，环境就绪。

## 编辑器推荐

- **VS Code** + 扩展 `P4 Analyzer`：语法高亮、跳转、类型检查
- **Vim / Neovim**：`vim-p4`（@mpu/vim-p4）提供基础高亮
- **JetBrains**：暂无官方插件，可配合 `Custom Language` 配置

## 目录约定（本教程通用）

后续章节会用到如下约定目录：

```text
examples/<编号>-<名字>/
├── README.md                  # 背景、拓扑、预期效果
├── <名字>.p4                   # P4 源码
├── build.sh                   # 封装 p4c 调用
├── topology/                  # Mininet 拓扑与启动脚本
├── runtime/                   # 控制平面下发脚本（表项 JSON / Python）
└── test/                      # 验证脚本（ping、iperf、pcap）
```
