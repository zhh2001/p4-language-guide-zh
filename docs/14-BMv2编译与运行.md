# 14 · BMv2 编译与运行

> 本章目标：把 `p4c` → `simple_switch` → Mininet 这条最主流的工具链走通。看完你能把任何一个 P4 程序部署到软件交换机里做实验。

## 14.1 工具链全景

```text
  hello.p4                     ←  你写的源码
     │
     │ p4c-bm2-ss
     ▼
  hello.json  hello.p4info.txt
     │                │
     │                └── 给控制平面用（P4Runtime 必需）
     │
     │ simple_switch 加载
     ▼
  数据平面（BMv2 软件交换机）
     ▲
     │ 流量由 Mininet / veth 提供
     │
 主机 h1, h2, ...
```

三个关键组件：

| 组件 | 作用 |
| ---- | ---- |
| `p4c-bm2-ss` | P4 编译器 BMv2 后端 |
| `simple_switch` | BMv2 的软件交换机（不带 gRPC） |
| `simple_switch_grpc` | 带 P4Runtime gRPC 的版本 |
| `simple_switch_CLI` | 通过 Thrift 下发表项的 CLI |
| Mininet | 虚拟拓扑工具 |

## 14.2 编译

### 14.2.1 基本命令

```bash
p4c-bm2-ss \
    --target bmv2 \
    --arch v1model \
    -o hello.json \
    --p4runtime-files hello.p4info.txt \
    hello.p4
```

关键参数：

| 参数 | 说明 |
| ---- | ---- |
| `--target bmv2` | 目标是 BMv2 |
| `--arch v1model` | 使用 V1Model 架构 |
| `-o out.json` | 生成给 BMv2 加载的 JSON |
| `--p4runtime-files out.p4info.txt` | 生成 P4Info（P4Runtime 用） |
| `-I dir` | include 路径 |
| `-D MACRO` | 预处理宏 |
| `--Werror` | 把警告当错误 |

### 14.2.2 常见编译错误

| 报错 | 常见原因 |
| ---- | -------- |
| `Could not find architecture file 'v1model.p4'` | p4c 没装好；检查 `$P4C_INCLUDE` |
| `Cannot cast from 'bit<X>' to 'bit<Y>'` | 位宽不匹配；手动转换 |
| `Expected member 'ingress_port' of type 'in'` | 方向写错；参考架构定义 |
| `Reserved keyword 'table' used as identifier` | 用了保留字做标识符 |

### 14.2.3 用 Makefile 管理

仓库 [`examples/03-ipv4-router/Makefile`](../examples/03-ipv4-router/Makefile) 给了一个参考：

```makefile
P4C ?= p4c-bm2-ss
ARGS ?= --target bmv2 --arch v1model
SRC  := router.p4

build: $(SRC:.p4=.json)

%.json: %.p4
    $(P4C) $(ARGS) -o $@ --p4runtime-files $(basename $<).p4info.txt $<

clean:
    rm -f *.json *.p4info.txt
```

## 14.3 启动 BMv2

### 14.3.1 `simple_switch`

```bash
sudo simple_switch \
    --log-console \
    -i 1@veth1 -i 2@veth3 \
    hello.json
```

参数解释：

- `--log-console` → 把内部日志打到屏幕（调试必开）
- `-i <port>@<iface>` → 把 Linux 接口 `veth1` 绑到 BMv2 端口 `1`
- 最后一个参数 → 加载的 P4 JSON

### 14.3.2 `simple_switch_grpc`

如果要用 P4Runtime，启 GRPC 版本：

```bash
sudo simple_switch_grpc \
    --log-console \
    -i 1@veth1 -i 2@veth3 \
    --no-p4 \
    -- --grpc-server-addr 127.0.0.1:50051
```

注意 `--no-p4` 先启动空白交换机——随后控制平面通过 P4Runtime 推送 JSON + 表项。这是 P4Runtime 推荐的工作流。

### 14.3.3 日志等级

```bash
--log-level debug          # trace/debug/info/warn/error
--log-file /tmp/bmv2.log
--log-flush                # 立即 flush 到 disk
```

### 14.3.4 Nanomsg 消息总线

BMv2 在端口 9090 发送 Nanomsg 事件（parser error、table miss、drop）。搭配 `bm_nanomsg_events` 脚本可实时监控：

```bash
simple_switch --notifications-addr ipc:///tmp/bmv2-0-notifications.ipc ...
```

## 14.4 创建网络接口

### 14.4.1 方法一：手工 veth（最轻量）

```bash
for i in 0 1 2 3; do
    sudo ip link add veth$((i*2))     type veth peer name veth$((i*2+1))
    sudo ip link set veth$((i*2))     up
    sudo ip link set veth$((i*2+1))   up
    sudo ethtool -K veth$((i*2))     tx off rx off sg off
    sudo ethtool -K veth$((i*2+1))   tx off rx off sg off
done
```

然后 `-i 1@veth0 -i 2@veth2 ...` 绑到 BMv2。

### 14.4.2 方法二：Mininet

标准做法，仓库每个示例都有 `topology/` 目录里的 topology.py：

```python
# examples/02-l2-switch/topology/topology.py
from mininet.net  import Mininet
from mininet.node import Host
from p4utils.mininetlib.network_API import NetworkAPI

net = NetworkAPI()
net.setLogLevel('info')

net.addP4Switch('s1', cli_input='runtime/s1-commands.txt')
net.setP4Source('s1', 'l2_switch.p4')

net.addHost('h1')
net.addHost('h2')

net.addLink('h1', 's1', port2=1)
net.addLink('h2', 's1', port2=2)

net.l2()
net.enablePcapDumpAll()
net.enableLogAll()
net.startNetwork()
```

借助 `p4-utils`（[p4utils](https://github.com/nsg-ethz/p4-utils)）可以一行起一个带 P4 的 Mininet 网络。

## 14.5 下发表项

### 14.5.1 `simple_switch_CLI`（Thrift）

编译时生成 `hello.json`，启动 BMv2 后可以用 CLI 直接下发：

```bash
simple_switch_CLI --thrift-port 9090 < runtime/s1-commands.txt
```

命令示例（`s1-commands.txt`）：

```text
table_add ipv4_lpm set_nhop 10.0.1.1/32 => 10.0.1.1 1
table_add ipv4_lpm set_nhop 10.0.2.0/24 => 10.0.2.2 2
table_set_default ipv4_lpm drop
counter_reset pkt_count
```

常用 CLI 命令：

| 命令 | 作用 |
| ---- | ---- |
| `table_add T A K => V` | 加表项 T 中匹配 K 的动作是 A(V) |
| `table_delete T <handle>` | 删除表项 |
| `table_set_default T A V` | 设默认动作 |
| `counter_read T <idx>` | 读计数器 |
| `register_read R <idx>` | 读寄存器 |
| `mc_mgrp_create <id>` / `mc_node_create` | 配置多播组 |

### 14.5.2 P4Runtime（推荐）

标准的 gRPC API。详见 [15 · P4Runtime 控制平面](./15-P4Runtime控制平面.md)。

### 14.5.3 P4Runtime Shell

交互式命令行，适合调试：

```bash
pip install p4runtime-shell

python3 -m p4runtime_shell \
    --grpc-addr 127.0.0.1:50051 \
    --device-id 0 \
    --election-id 0,1 \
    --config hello.p4info.txt,hello.json
```

进入 Python REPL：

```python
>>> te = table_entry['MyIngress.ipv4_lpm'](action='MyIngress.set_nhop')
>>> te.match['hdr.ipv4.dstAddr'] = ('10.0.1.0', 24)
>>> te.action['port'] = '1'
>>> te.action['nh'] = '10.0.1.1'
>>> te.insert()
```

## 14.6 抓包调试

### 14.6.1 tcpdump

```bash
sudo tcpdump -ni s1-eth1 -w s1-eth1.pcap
```

### 14.6.2 Wireshark

Wireshark 官方带 **P4 解析插件**（需自己编译，或使用 `nslookup` 之类的标准解析）。更方便：

- 用 `pcap` 文件 + Wireshark GUI 观察流量
- P4 自定义头部，可写 Lua 解析脚本

## 14.7 常用一键脚本

仓库每个示例下的 `run.sh` 大致结构：

```bash
#!/usr/bin/env bash
set -euo pipefail

HERE=$(cd "$(dirname "$0")" && pwd)
cd "$HERE"

# 1. 编译
p4c-bm2-ss --target bmv2 --arch v1model \
    -o prog.json \
    --p4runtime-files prog.p4info.txt \
    prog.p4

# 2. 起 Mininet
sudo python3 topology/topology.py

# 3. 在 Mininet CLI 里：
#    mininet> h1 ping h2
```

## 14.8 性能提示

BMv2 是教学级软件交换机，**不要** 期望拿它测吞吐：

- 单核 ~ 10k pps
- 无 SIMD、无硬件卸载
- 调试用 `--log-console` 会进一步拖慢

想测性能：

- 用 `simple_switch` 加 `--dpdk`（实验性）
- 或直接上 DPDK 目标（p4c-dpdk）
- 最终上真实硬件（Tofino / Intel N6000 / Mellanox 等）

## 14.9 常见问题

| 问题 | 原因 | 解决 |
| ---- | ---- | ---- |
| BMv2 启动后不收包 | veth tx offload 没关 | `ethtool -K vethX tx off` |
| CLI `table_add` 报 "table not found" | FQN 错 | 用 `show_tables` 先看清名字 |
| P4Runtime 连接拒绝 | 用了 `simple_switch` 而非 gRPC 版 | 改用 `simple_switch_grpc` |
| 报文被交换机吃掉 | Ingress drop 了 | 看日志里的 `set_metadata egress_spec=511` |
| ICMP 校验和出错 | 忘了 `update_checksum` | ComputeChecksum 补上 |

## 14.10 本章小结

- 工具链：`p4c-bm2-ss` → `simple_switch(_grpc)` → Mininet
- 下发表项两种路线：**CLI（Thrift）** 和 **P4Runtime（gRPC）**
- 调试三板斧：BMv2 日志 + `tcpdump` + P4Runtime Shell
- BMv2 定位是 **学习 / 测试**，性能瓶颈是单核 ~ 10k pps
