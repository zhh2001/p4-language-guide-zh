# 04 · 语法基础

> 本章目标：把 P4_16 的语言骨架——预处理、标识符、字面量、运算符、语句——讲清楚。学完后你能读懂任何 P4 程序的"外壳"，但还不包括类型系统和具体块（Parser/Control/Deparser）的细节。

## 4.1 预处理器

P4 编译器内置了 C 预处理器的子集：

- `#define` / `#undef`
- `#if` / `#else` / `#endif` / `#ifdef` / `#ifndef` / `#elif`
- `#include`

`#include` 和 C 一样，尖括号找系统库，双引号找本地文件：

```p4
#include <core.p4>     // 系统路径（一般是 p4c 安装目录）
#include "my_utils.p4" // 当前目录
```

### 核心库必须包含

```p4
#include <core.p4>
```

`core.p4` 里定义了 `packet_in`、`packet_out` 等基础 extern，以及 `NoAction`、`exact/ternary/lpm` 等 match_kind。**任何 P4 程序都必须包含它**。

## 4.2 标识符与保留字

- 由字母、数字、下划线组成，不能以数字开头
- `_`（单个下划线）是保留符号，表示"不关心这个值"

```text
abstract     action       apply        bit          bool
const        control      default      else         enum
error        extern       exit         false        header
header_union if           in           inout        int
list         match_kind   package      parser       out
return       select       state        string       struct
switch       table        this         transition   true
tuple        type         typedef      value_set    varbit
verify       void
```

这些词都不能拿来做变量/类型名。

## 4.3 命名约定

P4 规范推荐（但不强制）以下命名风格：

| 元素 | 风格 | 例 |
| ---- | ---- | -- |
| 内置类型 | 全小写 | `int<16>` |
| 自定义类型 | 首字母大写 | `IPv4Address`、`Ethernet_h` |
| 类型变量（泛型） | 全大写 | `parser P<H, IH>()` |
| 变量 | 首字母小写 | `ipv4header` |
| 常量 | 全大写 | `CPU_PORT` |
| error / enum 成员 | 驼峰 | `PacketTooShort` |

读 p4lang 官方示例时，你会看到 `_h`/`_t` 后缀被大量使用（`Ethernet_h`、`metadata_t`），建议沿用。

## 4.4 注释

和 C 一模一样：

```p4
// 单行注释

/* 多行
   注释 */
```

## 4.5 字面量

### 4.5.1 布尔

只有两个值：`true`、`false`。

### 4.5.2 整数字面量

P4 的整数字面量支持 **多进制 + 位宽 + 有无符号** 三个维度，是一个需要特别理解的特性。

**进制前缀**

| 前缀 | 含义 |
| ---- | ---- |
| `0x` 或 `0X` | 十六进制 |
| `0o` 或 `0O` | 八进制 |
| `0d` 或 `0D` | 十进制 |
| `0b` 或 `0B` | 二进制 |
| 无 | 十进制 |

> [!WARNING]
> **与 C 的关键区别**
> C 把 `0377` 解释为八进制（= 255），**P4 当作十进制 377**。想表达八进制请显式写 `0o377`。

**宽度/符号前缀**

| 前缀 | 含义 |
| ---- | ---- |
| `Nw`  | 无符号、宽度 `N` |
| `Ns`  | 有符号、宽度 `N` |

组合示例：

```p4
32w255           // bit<32>，值 255
32w0d255         // 同上
32w0xFF          // 同上
32s0xFF          // int<32>，值 255
8w0b10101010     // bit<8>，值 170
8w0b_1010_1010   // 同上（用 _ 分隔提高可读性）
8w170            // 同上
8s0b1010_1010    // int<8>，值 -86（补码解读）
16w0377          // bit<16>，值 377（不是 255！）
16w0o377         // bit<16>，值 255
```

### 4.5.3 下划线分隔符

数字中可以插入任意多的下划线提高可读性：

```p4
48w0x00_11_22_33_44_55    // 以太网 MAC 地址字面量
```

### 4.5.4 字符串

双引号包围，反斜杠转义：

```p4
"simple string"
"string \" with \" embedded \" quotes"
```

> [!TIP]
> P4 程序中 **只能出现字符串字面量**——不能声明 `string` 类型的变量，只能作为无方向参数传入函数。字符串的主要用途是注解 `@name("...")`、`static_assert("...")`。

## 4.6 运算符速查

| 类别 | 运算符 |
| ---- | ------ |
| 算术 | `+` `-` `*` `/` `%` |
| 位运算 | `&` `\|` `^` `~` `<<` `>>` |
| 比较 | `==` `!=` `<` `<=` `>` `>=` |
| 逻辑 | `&&` `\|\|` `!` |
| 位串拼接 | `++` |
| 位切片 | `[H:L]` |
| 条件 | `? :` |
| 掩码（集合） | `&&&` |
| 范围（集合） | `..` |

P4 **不支持** 自增 / 自减（`++i`、`i++`）、**不支持** `=` 之外的复合赋值（C 里的 `+=`/`-=` 等），**不支持** 浮点，**不支持** 逗号表达式。

### 4.6.1 位切片

```p4
bit<32> ip;
bit<8>  first_byte  = ip[31:24];   // 取最高字节
bit<8>  second_byte = ip[23:16];
```

索引是 **闭区间** 且 **高位在前**。

### 4.6.2 位串拼接

```p4
bit<16> a = 0xAA00;
bit<8>  b = 0xFF;
bit<24> c = a ++ b;   // 0xAA00FF
```

## 4.7 语句

每条语句必须以 `;` 结尾。P4 里的语句种类比 C 少，而且不同上下文（Parser / Control / Action）对可用的语句类型有限制。

### 4.7.1 赋值

```p4
x = 3;
hdr.ipv4.ttl = hdr.ipv4.ttl - 1;
```

**extern 类型不支持赋值**（它们是外部资源，得通过方法调用）。

### 4.7.2 块语句

```p4
{
    bit<8> tmp = 1;
    tmp = tmp + 1;
}
```

块内声明的变量只在块内可见。

### 4.7.3 空语句

一个单独的 `;`，什么也不做——通常用来占位。

### 4.7.4 条件语句

```p4
if (hdr.ipv4.ttl == 0) {
    drop_action();
} else if (hdr.ipv4.ttl == 1) {
    send_to_cpu();
} else {
    hdr.ipv4.ttl = hdr.ipv4.ttl - 1;
}
```

条件必须是 `bool` 类型——**不能像 C 那样用整数当布尔**。

### 4.7.5 switch 语句

**只能在 `control` 块内使用**。支持两种 switch：

**① 基于 `table.apply().action_run`**

用于判断某张表实际命中了哪个 action：

```p4
switch (ipv4_lpm.apply().action_run) {
    forward: { /* 命中 forward */ }
    drop:    { /* 命中 drop */ }
    default: { /* 未命中或命中其他 */ }
}
```

**② 基于整数 / 枚举 / error**

```p4
switch (hdr.ethernet.etherType) {
    0x0800: { parse_ipv4(); }
    0x0806: { parse_arp(); }
    default: { drop(); }
}
```

> [!WARNING]
> **关键差异**：P4 的 switch **不** 自动 fall-through。
> - 若 label 后 **有 `{ }` 块**：执行完就跳出 switch
> - 若 label 后 **没有块**：自动 fall-through 到下一个 label
>
> 这与 C 语言完全相反。

```p4
switch (x) {
    1:       // fall-through 到 2
    2: { /* 1 或 2 */ }
    3: { /* 仅 3，不 fall-through */ }
}
```

### 4.7.6 `return`

- 可在 `action`、**有返回值的函数**、`control.apply` 内使用
- **不能** 在 `parser` 中使用（parser 用 `transition`）

### 4.7.7 `exit`

- 立即终止 **当前 control 块及其所有调用者**
- 不能在 `parser` 和 `function` 里使用

### 4.7.8 `transition`（Parser 专属）

```p4
state parse_ethernet {
    packet.extract(hdr.ethernet);
    transition select(hdr.ethernet.etherType) {
        0x0800: parse_ipv4;
        default: accept;
    }
}
```

详情见 [06 · Parser 解析器](./06-Parser解析器.md)。

## 4.8 参数方向修饰符

P4 函数/动作/块的每个参数都必须声明方向（或保持无方向）：

| 方向 | 语义 | 类比 |
| ---- | ---- | ---- |
| `in`    | 输入；被调用方只能读 | C++ `const T&` |
| `out`   | 输出；被调用方必须赋值，调用方拿到结果 | C `T*`（out param） |
| `inout` | 先输入后输出，被调用方可读可改 | C++ `T&` |
| 无方向  | 编译期常量，行为同 `in` | template non-type parameter |

只有 `in` 和无方向参数可以有 **默认值**：

```p4
extern void f(in bit a, in bit<3> b = 2, in bit<5> c);

void g() {
    f(a = 1, b = 2, c = 3);  // ok
    f(a = 1, c = 3);         // ok，b 用默认值
    f(1, 2, 3);              // ok，按位置
    f(1, 3);                 // ❌ 不能省略非默认参数的位置
}
```

### 可选参数 `@optional`

```p4
package pipeline(/* ... */);
package switch(pipeline first, @optional pipeline second);

pipeline(/* ... */) ingress;
switch(ingress) main;   // 只实例化了单级流水线
```

可选参数不能设置默认值。

## 4.9 尾随逗号

P4 允许列表最后一项后保留逗号，方便增删和配合预处理：

```p4
enum E {
#if SUPPORT_A
    a,
#endif
    b,
    c,   // 合法
}
```

## 4.10 名称解析

P4 有一个顶层匿名命名空间。用 `.标识符` 强制在顶层解析：

```p4
const bit<32> x = 2;
control c() {
    int<32> x = 0;
    apply {
        x = x + (int<32>).x;   // .x 是顶层的 bit<32>
                               // 左侧的 x 是 control 内的 int<32>
    }
}
```

标识符查找顺序：**当前作用域 → 外层作用域 → …… → 顶层**。

## 4.11 函数调用

调用时可以按位置，也可以按名称，但 **不能混用**：

```p4
extern void f(in bit<32> x, out bit<16> y);
bit<32> xa = 0;
bit<16> ya;

f(xa, ya);           // 按位置
f(x = xa, y = ya);   // 按名称
f(y = ya, x = xa);   // 按名称，顺序任意
f(x = xa, ya);       // ❌ 混用
f(x = xa);           // ❌ 缺参数
f(x = xa, y = 0);    // ❌ out 参数必须是 lvalue
```

## 4.12 函数声明

函数只能在 **最外层** 声明，且参数必须有方向：

```p4
bit<32> max(in bit<32> a, in bit<32> b) {
    return (a > b) ? a : b;
}
```

> [!WARNING]
> P4 **不支持递归函数**——也不支持间接递归。这是硬件静态展开的约束。

## 4.13 常量声明

```p4
const bit<32> CPU_PORT = 0xFD;

struct Version { bit<32> major; bit<32> minor; }
const Version v = { 1, 0 };
```

常量必须在声明时初始化，生命周期为整个程序。

## 4.14 `typedef` 与 `type`

两种类型别名：

```p4
typedef bit<32> IPv4Addr;  // 别名，原类型可以互换
type    bit<32> NodeId;    // 新类型，需要显式转换
```

**区别**：`typedef` 是透明别名；`type` 创造一个"名义类型"——编译器会阻止它和底层类型隐式互换。后者常用于让编译器帮你检测"把端口号当 IP 用"这种错。

## 4.15 显式与隐式类型转换

### 允许的隐式转换（仅这两种）

1. `int` → 固定宽度整数（`bit<N>` / `int<N>`）
2. 带底层类型的 enum → 该底层类型

其他一律要显式写 `(T)e` 才能转。

### 常见显式转换规则

| 源 → 目标 | 行为 |
| --------- | ---- |
| `bit<1>` ↔ `bool` | 0⇌false, 1⇌true |
| `int<W>` → `bit<W>` | 位保持不变，符号位当数值位 |
| `bit<W>` → `int<W>` | 位保持不变，最高位变符号位 |
| `bit<W>` → `bit<X>` | W>X 截断，W<X 零扩展 |
| `int<W>` → `int<X>` | W>X 截断，W<X 符号扩展 |
| `bit<W>` → `int`（无限） | 值不变 |
| `int` → `bit<W>` | 补码截断到 W 位 |

### "看起来能编但其实不行"的表达式

```p4
bit<8>  x;
bit<16> y;
int<8>  z;
```

| 错误写法 | 原因 | 正确写法 |
| -------- | ---- | -------- |
| `x + y` | 位宽不同 | `(bit<16>)x + y` |
| `x + z` | 符号不同 | `x + (bit<8>)z` |
| `(int<8>)y` | 不能同时改符号和位宽 | `(int<8>)(bit<8>)y` |
| `y + z` | 位宽+符号都不同 | `(int<16>)y + (int<16>)z` |
| `x << z` | 位移量不能是有符号数 | `x << (bit<8>)z` |
| `1 << x` | 对任意精度整数做位运算 | `32w1 << x` |
| `~1` | 同上 | `~32w1` |

## 4.16 本章小结

- 预处理、标识符、字面量基本沿用 C 的直觉，**但 `0377` 是十进制 377**
- 整数字面量语法 `<W>w/s<value>` 让位宽和符号变得显式
- 语句种类少，且 **不同块能用的语句不同**；parser 不能 `return`，function 不能 `switch`
- switch **不自动 fall-through**，这和 C 相反
- 参数方向是强制的：`in` / `out` / `inout` / 无方向
- 隐式转换极少，别嫌啰嗦——这是 P4 避开 C 家族"无声 bug"的有意设计
