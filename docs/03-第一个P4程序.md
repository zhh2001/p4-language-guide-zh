# 03 · 第一个 P4 程序：Hello P4

> 本章目标：花 15 分钟写一个 **最小可运行的 P4 程序**，在 BMv2 上收到报文、原路返回。学完你会对 P4 程序的"骨架"有清晰印象。

我们要做的事情非常简单：**交换机收到任何报文，把它从它进来的那个端口再发回去**（报文反射）。这听起来没什么用，但它包含了完整的 P4 流水线要素：Parser、Ingress、Egress、Deparser、Architecture 实例化。

## 3.1 目标拓扑

```text
        ┌──────────────────────┐
 h1 ────┤  s1  (BMv2)          ├──── h2
 port 1 │    Hello P4 程序     │ port 2
        └──────────────────────┘
```

h1 ping h2 时，报文到达 s1，被 P4 程序把出端口设置为"进端口"，因此包会原路返回到 h1。h1 和 h2 彼此不通——这就证明了我们的 P4 程序确实生效了。

## 3.2 完整代码

完整源码在 [`examples/01-hello/hello.p4`](../examples/01-hello/hello.p4)。这里我们边写边讲。

```p4
// examples/01-hello/hello.p4
#include <core.p4>
#include <v1model.p4>
```

- `core.p4` 是 P4 核心库，提供 `packet_in.extract()`、`verify()` 等原语。**所有 P4 程序必写**。
- `v1model.p4` 是我们选择的 **架构描述**——BMv2 `simple_switch` 使用的架构。

### 3.2.1 定义数据类型

```p4
struct metadata   { }
struct headers    { }
```

因为我们不需要解析任何报头，也不需要暂存元数据，两个 struct 先留空。

> **为什么还要写空 struct？**
> 因为 `V1Switch` 架构的接口要求 Parser、Control 等块接收这两个参数。即使我们不用，"形状"也必须对得上。

### 3.2.2 Parser

```p4
parser MyParser(packet_in        packet,
                out   headers    hdr,
                inout metadata   meta,
                inout standard_metadata_t standard_metadata) {
    state start {
        transition accept;  // 直接接受，不解析任何报头
    }
}
```

`start` 是必需的起始状态。我们没有调用 `extract`，相当于**放行原始字节流**。

### 3.2.3 Checksum Verify（占位）

V1Model 要求在 Ingress 前后都提供校验和控制块；即便不用也要声明：

```p4
control MyVerifyChecksum(inout headers hdr, inout metadata meta) {
    apply { }
}
```

### 3.2.4 Ingress —— 真正的转发逻辑

```p4
control MyIngress(inout headers hdr,
                  inout metadata meta,
                  inout standard_metadata_t standard_metadata) {
    apply {
        // 把出端口设置为进端口，实现"报文反射"
        standard_metadata.egress_spec = standard_metadata.ingress_port;
    }
}
```

`standard_metadata` 是 V1Model 给每个报文配的"随行元数据"——里面有 `ingress_port`、`egress_spec`（你要发往的端口）、`packet_length` 等字段。

### 3.2.5 Egress 与 ComputeChecksum（占位）

```p4
control MyEgress(inout headers hdr,
                 inout metadata meta,
                 inout standard_metadata_t standard_metadata) {
    apply { }
}

control MyComputeChecksum(inout headers hdr, inout metadata meta) {
    apply { }
}
```

### 3.2.6 Deparser

```p4
control MyDeparser(packet_out packet, in headers hdr) {
    apply { }   // 没解析任何报头，自然也不需要 emit
}
```

### 3.2.7 实例化顶层包

```p4
V1Switch(
    MyParser(),
    MyVerifyChecksum(),
    MyIngress(),
    MyEgress(),
    MyComputeChecksum(),
    MyDeparser()
) main;
```

`V1Switch` 就是 v1model 架构定义的顶层包类型，必须有一个叫 `main` 的实例化结果，编译器才知道怎么把六大块"串"起来。

## 3.3 编译

```bash
cd examples/01-hello

p4c-bm2-ss --target bmv2 --arch v1model \
    -o hello.json \
    --p4runtime-files hello.p4info.txt \
    hello.p4
```

成功后会得到：

- `hello.json` —— BMv2 能直接加载的 JSON 格式运行时配置
- `hello.p4info.txt` —— 给控制平面看的"表/动作清单"（本例没有表，文件很小）

## 3.4 启动 BMv2 和 Mininet

我们用 Mininet 搭一个最小拓扑：1 个 BMv2 交换机 + 2 个主机。

```bash
# 仓库里提供了一键脚本
sudo bash examples/01-hello/run.sh
```

脚本内部大致做这些事：

```bash
# 启动 BMv2
simple_switch \
    --log-console \
    -i 1@s1-eth1 -i 2@s1-eth2 \
    hello.json &

# 启动 Mininet（另一个终端）
sudo mn --custom topology.py --topo mytopo \
        --switch bmv2 --controller none
```

> [!TIP]
> 不想自己装 Mininet？可以直接用 `veth` + `nsenter` 做手工拓扑。仓库脚本里有替代实现，看 `run-veth.sh`。

## 3.5 验证

在 Mininet CLI 里：

```text
mininet> h1 ping h2
PING 10.0.0.2 (10.0.0.2) 56(84) bytes of data.
From 10.0.0.1 icmp_seq=1 Destination Host Unreachable
From 10.0.0.1 icmp_seq=2 Destination Host Unreachable
```

h1 ping h2 **不通**——因为我们把所有报文从原端口反射回去了。但：

```text
mininet> h1 tcpdump -i h1-eth0 -nn &
mininet> h1 ping -c 1 10.0.0.2
```

你会看到 h1 自己发出去的 ICMP Request 又**从同一个接口收回来了**（以 ICMP echo request 形式，因为 s1 原样反射了）。证明 P4 程序确实生效。

## 3.6 流水线走位回顾

一次完整的报文处理旅程：

```text
报文 ──► [Parser.start: transition accept]     ──► 报头结构为空
        [MyVerifyChecksum: noop]
        [MyIngress: egress_spec = ingress_port] ──► 告诉架构从原端口发出
        [Traffic Manager: 内建功能，按 egress_spec 调度]
        [MyEgress: noop]
        [MyComputeChecksum: noop]
        [MyDeparser: 无 emit，原样发出]         ──► 出端口
```

每一步都在做最少的工作，但完整走了一遍 PISA 流水线。

## 3.7 你学到了什么

- P4 程序 = Parser + Control 块若干 + Deparser + 架构实例化
- **V1Switch** 要求六大块：Parser、VerifyChecksum、Ingress、Egress、ComputeChecksum、Deparser
- 流水线级别的数据靠 `standard_metadata` 在块之间传递
- `p4c-bm2-ss` 把源码编成 `hello.json`，BMv2 加载后即可运行

## 3.8 常见报错

| 报错 | 原因 | 解决 |
| ---- | ---- | ---- |
| `standard_metadata_t is undefined` | 没 include v1model.p4 | 检查顶部 `#include` |
| `undefined reference to main` | 没实例化顶层包 | 底部加 `V1Switch(...) main;` |
| `simple_switch: error while loading shared libraries` | BMv2 依赖缺失 | 重新安装或用 Docker |
