# 13 · 注解与高级特性

> 本章目标：把一些"不是核心，但写真代码一定会碰到"的特性补齐——注解（`@name`/`@hidden`/`@atomic` 等）、`static_assert`、泛型、构造函数调用、可选参数。

## 13.1 注解（Annotation）

注解是"**不改变语法，又能附加信息**"的扩展机制。用 `@` 开头。

### 13.1.1 两种形式

- **非结构化**（`()`）：主体可选
- **结构化**（`[]`）：主体必选

```p4
@my_anno(1)       table T { /* ... */ }   // 非结构化
@my_anno[2]       table U { /* ... */ }   // 结构化
```

### 13.1.2 同元素上的命名规则

| 组合 | 是否允许 |
| ---- | -------- |
| 同元素两个同名 **非结构化** | ✅ |
| 同元素两个同名 **结构化** | ❌ |
| 同元素 **结构化 + 非结构化 同名** | ❌ |

### 13.1.3 非结构化注解的主体

```p4
@note("legacy, see RFC 1234")
```

主体是几乎任意字符串，语义由编译器/目标自行解释。

### 13.1.4 结构化注解的主体

三种合法形式：

```p4
@Empty[]                                   // 空
@MixedList[1, "hello", true, 1==2]         // 表达式列表
@Labels[short="L1", long="Label One"]      // 键值对
```

**不允许**键值对和表达式列表混用：

```p4
@IllegalMixing[key=4, 5]   // ❌
```

**同一结构化注解里键名不能重复**：

```p4
@DupKey[k1=4, k1=5]   // ❌
```

## 13.2 预定义注解速查表

| 注解 | 说明 |
| ---- | ---- |
| `@name("...")` | 控制平面看到的名字 |
| `@hidden` | 从控制平面隐藏此实体 |
| `@atomic` | 原子执行整块代码 |
| `@optional` | 构造参数可选 |
| `@tableonly` | action 只能出现在表项里 |
| `@defaultonly` | action 只能出现在默认动作里 |
| `@deprecated("msg")` | 被使用时编译器打印警告 |
| `@noWarn("类别")` | 抑制指定类别的警告 |
| `@match(kind)` | 指定 value_set 字段的 match_kind |

### 13.2.1 `@name` —— 改控制平面可见名字

P4 的完全限定名（FQN）默认由嵌套路径决定，比如：

```p4
control c(/* ... */)() {
    table t1 { /* ... */ }
}
c() c_inst;
```

这里 `c_inst.t1` 是完全限定名。控制平面写表项时要用它。

手动改名：

```p4
control c(/* ... */)() {
    @name("my_custom_table") table t1 { /* ... */ }
}
```

现在 FQN 变成 `c_inst.my_custom_table`。

#### 相对 vs 绝对

- `@name("t1")` —— 相对：拼在 scope 前面
- `@name(".t1")` —— 绝对：直接用 `t1`（注意前导点）

#### 约束

- 一个实体最多一个 `@name` 或 `@hidden`
- 解析后，**每个控制平面名字只能指向一个实体**；否则控制平面无法消歧

### 13.2.2 `@hidden` —— 从控制平面彻底隐藏

```p4
@hidden table internal_debug_tbl { /* ... */ }
```

适合实现细节表——编译器生成，但控制平面看不到，也就改不了。

### 13.2.3 `@atomic` —— 原子块

```p4
@atomic {
    register_read(c, index);
    c = c + 1;
    register_write(index, c);
}
```

保证这整块代码对 **同一报文** 的其他处理互斥。硬件上可能编译成 `lock` 或原子操作序列。

### 13.2.4 `@tableonly` / `@defaultonly`

精确控制 action 的可见性——见 [08.5.2 节](./08-匹配动作表.md)。

### 13.2.5 `@deprecated`

```p4
@deprecated("Use 'check_v2' instead")
extern Checker { /* ... */ }
```

任何使用 `Checker` 的地方，编译器都会打印 "Use 'check_v2' instead"。

### 13.2.6 `@noWarn`

抑制编译器指定类别的警告：

```p4
@noWarn("unused")
action helper() { /* ... */ }      // 不会被警告 "未使用"

@noWarn("duplicate_priorities")
entries = { /* 有意的重复优先级 */ }
```

### 13.2.7 `@optional`

让构造参数可缺省（**不是**函数参数，注意区分）：

```p4
package Sw(pipeline first, @optional pipeline second);
Sw(ingress) main;             // 仅一级流水线
```

可选参数 **不能有默认值**——缺省就是没传。

### 13.2.8 `@match(match_kind)`

用于 `value_set`，指定字段用什么匹配：

```p4
value_set<bit<16>>(4) udp_ports;

parser P(packet_in pkt, ...) {
    state start {
        transition select(hdr.udp.dstPort) {
            udp_ports: known_port;
            default:   other;
        }
    }
}
```

默认 `exact`。要 ternary 的话加 `@match(ternary)`。

## 13.3 `static_assert`

编译期断言。核心库里定义：

```p4
extern bool static_assert(bool check, string message);
extern bool static_assert(bool check);
```

### 13.3.1 用法

```p4
const bool _chk = static_assert(
    V1MODEL_VERSION >= 20180101,
    "Requires v1model version >= 20180101");
```

如果 `check` 为 false，编译器立即停止并打印 message。

### 13.3.2 典型用途

- 检查架构版本
- 检查配置宏的合法性
- 防止用户改动结构体顺序破坏对齐

## 13.4 泛型（Generics）

P4_16 支持在以下位置使用类型参数：

- `extern`
- `parser` 类型（**不能**在 parser 实现里泛型）
- `control` 类型
- `package`
- `function`

### 13.4.1 声明

```p4
extern T identity<T>(in T x);

parser Parser<H, M>(packet_in b,
                    out   H     hdr,
                    inout M     meta,
                    inout standard_metadata_t std);
```

### 13.4.2 实例化

通常编译器能推断；推断不出的时候显式传：

```p4
Switch<bit<32>>(P(), Pipe1()) main;
```

### 13.4.3 泛型的常见坑

- **parser / control 的实现不能泛型**；只有它们的类型声明可以
- 递归泛型（`T` 自身出现在 `T` 的参数里）不允许

## 13.5 构造函数调用

带有构造函数的对象：`extern` / `parser` / `control` / `package`。

```p4
extern ActionProfile {
    ActionProfile(bit<32> size);   // 构造函数
}

table tbl {
    actions = { /* ... */ }
    implementation = ActionProfile(1024);   // 构造函数调用
}
```

### 13.5.1 构造参数 vs 运行时参数

```p4
control C(bit<32> size)  // 构造参数：编译期固化
         (inout headers hdr, /* 运行时参数 */) {
    /* ... */
}

C(1024) c_inst;
c_inst.apply(hdr, /* ... */);
```

- **构造参数**：在 `( )` 列表里，用于实例化
- **运行时参数**：在 `( )` 列表里（注意 P4 这里用了另一对括号），每次 apply 时传

## 13.6 可选参数与默认值

复习 [04.8 节](./04-语法基础.md#48-参数方向修饰符)：

- `in` 和 **无方向** 参数可以有默认值
- `@optional` 的构造参数可省略

```p4
extern void f(in bit a, in bit<3> b = 2, in bit<5> c);

f(a = 1, c = 3);      // ✅ b 默认
f(1, 3);              // ❌ 位置参数不能跳过 b
f(1, 2, 3);           // ✅
```

## 13.7 `value_set`

动态可配置的 "select 常量表"：

```p4
value_set<bit<16>>(8) valid_etypes;

state start {
    packet.extract(hdr.ethernet);
    transition select(hdr.ethernet.etherType) {
        valid_etypes: parse_rest;
        default:      accept;
    }
}
```

控制平面可以在运行时填充 `valid_etypes` 的成员——相当于"parser 里的轻量级表"。

## 13.8 `#include` 与组织代码

大型项目通常拆分成：

```text
my_project/
├── main.p4                ← 只写 main 实例化 + include
├── headers.p4             ← 所有 header + struct 定义
├── parsers.p4
├── tables/
│   ├── ipv4_fwd.p4
│   ├── ipv6_fwd.p4
│   └── acl.p4
└── controls.p4
```

```p4
// main.p4
#include <core.p4>
#include <v1model.p4>
#include "headers.p4"
#include "parsers.p4"
#include "controls.p4"

V1Switch(MyParser(), ..., MyDeparser()) main;
```

## 13.9 本章小结

- 注解是 P4 的扩展机制，分结构化 `[]` 和非结构化 `()`
- 三个最常用：`@name`（改控制平面名）、`@hidden`（隐藏实体）、`@atomic`（原子块）
- `@tableonly` / `@defaultonly` 精细控制 action 的用法
- `static_assert` 做编译期检查
- 泛型只能在 **类型声明 / extern / function** 里用
- 构造参数 vs 运行时参数，搞清楚括号配对
